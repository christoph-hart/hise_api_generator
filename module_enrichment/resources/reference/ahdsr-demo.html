AHDSR Envelope

AHDSR
EnvelopeModulator Generator
b.Modulators.AHDSR

A classic Attack-Hold-Decay-Sustain-Release envelope with adjustable curve shapes and per-voice modulation of all time and level parameters. All curve shapes use exponential coefficients (not lookup tables) - the AttackCurve parameter controls the attack target ratio (concave to convex), and DecayCurve controls both the decay and release curve shapes.

The envelope runs per-voice in polyphonic mode. In monophonic mode, a single shared state is used with optional retrigger behavior when new notes arrive. The sustain phase uses a per-buffer ramp when the sustain value changes via modulation, avoiding zipper noise.

Modulation Chains

Each time and level parameter has a corresponding VoiceStartModulator chain that scales the base value per-voice at note-on. These are evaluated once when the voice starts, not continuously.

Notes

The EcoMode parameter is vestigial - the setter is a no-op (source comment: "not needed anymore"). Downsampling is now controlled globally by HISE_CONTROL_RATE_DOWNSAMPLING_FACTOR (8x for instruments, 1x for effects).

The envelope state machine is shared with the scriptnode equivalent via ahdsr_base::state_base::tick(). All curve shapes use exponential coefficients computed from the curve parameter, not lookup tables.

Note that DecayCurve controls the curvature of both the decay and release phases - there is no separate release curve parameter.
