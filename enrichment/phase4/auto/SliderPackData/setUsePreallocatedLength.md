Configures a fixed-size preallocated buffer for the slider data. Once set, `setNumSliders()` calls up to the preallocated limit adjust a view into this fixed block without allocating new memory, preserving all existing values - including those beyond the current view that become visible again when the count increases. Pass 0 to disable preallocation and return to normal allocation behaviour. Call this immediately after creation for any slider pack whose count changes at runtime (e.g. variable-length step sequencers).
