Spawns an OS child process and streams its output line by line to a log callback. The log function receives three arguments: the task, a boolean `isFinished` flag, and a `data` value. During output, `data` is a String containing one line of text; on completion, `data` is an integer exit code. Check `isFinished` before using `data` to handle the type change correctly. Both stdout and stderr are merged into a single stream. The process respects `shouldAbort()` and is killed on abort.

> **Warning:** The timeout must exceed the longest interval between output lines from the child process. If the process goes silent for longer than the timeout, the thread may be force-killed with undefined behaviour. Use `setTimeOut()` with headroom above the expected output interval.
